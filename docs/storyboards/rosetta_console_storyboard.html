Rosetta Console

The operator cockpit for real-time multilingual market response. The storyboard highlights the primary console layout, live agent feed, and regime monitoring banner.

Mission

Provide a heads-up display that blends model outputs, tactical actions, and live status signals. Operators should get context first, with precision controls one tap away.

Storyboard · Main Canvas

00:00 — Launch Pad

Console boots into a dense overview of markets, active agents, and latest actions.

HUD overlays key metrics, timeline scrubbing available.
00:15 — Regime spotlight

Regime ticker anchors the top of the canvas. Operators can drill into the snapshot to audit the factors behind open / closed status.

Ticker must refresh without reflowing the main column.
00:45 — Multi-agent choreography

Feed transitions to show queued plans, side-channel translations, and recommended responses with confidence badges.

Operators can pin, diff, or send live.
Design note: The regime banner sits between the nav and scenario grid, bridging system telemetry with human decision making.

Storyboard · Support Modules

Command Stream

Chronological ledger of issued prompts and tool calls, grouped by mission thread.

Language Rosetta

Live translation matrix summarizing tone, risk, and commitments across locales.

Escalation Ladder

Pre-built movesets for the operator. Each shows playbook context, required approvals, and ETA.

Ops Chronicle

Automated post-mortem drafting. Surfaces stats, anomalies, and recommended follow-ups.

Operators can pop these modules into sidecars. Regime context persists in the ticker no matter the layout.
